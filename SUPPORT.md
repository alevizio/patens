# Where to ask, where to report

Patens is maintained solo by one person ([@alevizio](https://github.com/alevizio)).
To keep the issue tracker actionable and the conversation surface useful,
the channels are deliberately distinct:

## Found a bug?

→ **[Open an issue](https://github.com/alevizio/patens/issues/new?template=bug.md)**

Examples of bugs: a route 500s, a glyph fails to export, an audit fix
breaks geometry, a keyboard shortcut doesn't fire, a calculation is
wrong. Use the bug template — it asks for the route, the action, and
the actual vs. expected behaviour, which is what I need to reproduce.

## Have an idea for a new feature?

→ **[Open a feature request](https://github.com/alevizio/patens/issues/new?template=feature.md)**

For sharp, well-formed proposals (the problem, who it's for, why now).
If you're still thinking out loud, [Discussions](https://github.com/alevizio/patens/discussions)
is the better venue — see below.

## Have a question? Want to show off a font you drew? Half-formed idea?

→ **[Discussions](https://github.com/alevizio/patens/discussions)**

That's where Q&A lives, where people share what they've made with Patens,
and where feature ideas get sanded into something concrete before becoming
an issue. Discussions are conversational; issues are a punch list.

Some things that belong in Discussions:

- "How do I do X in the editor?" (likely the [Help page](https://patens.design/help) too)
- "I exported a font and the kerning looks weird — is this expected?"
- "I made a font — here it is" (show-and-tell)
- "What would it take to support [feature]?" (RFC-style)
- "Why doesn't Patens do X?" (philosophy / scope)

## Found a security vulnerability?

→ **Do NOT file a public issue.** Use one of:

- The private security advisory form: https://github.com/alevizio/patens/security/advisories/new
- Email: `security@patens.design`

Full disclosure policy in [SECURITY.md](./SECURITY.md).

## Want to contribute code?

→ [CONTRIBUTING.md](./CONTRIBUTING.md) covers the dev setup, the commit
conventions, and how to run the test suite. The [ROADMAP.md](./ROADMAP.md)
covers what's deliberately deferred and why — useful context before
opening a large PR.

## Response time

This is a solo project. I read everything but I don't always respond
quickly — security issues get the fastest turnaround. For most other
things, expect days, not hours.
