# Design Context — Font Studio

Last updated: 2026-05-19

This file is the single source of truth for design context. Read it before any
design work. If a decision conflicts with what's here, flag it before deviating.

---

## Users

**Two audiences, both first-class:**

1. **Solo type designers** — hobbyist or personal-foundry practitioners drawing
   their own typeface for the love of it. Already opinionated about type, often
   already familiar with Glyphs / FontLab / RoboFont. Use Font Studio for the
   "open browser, sketch a glyph, see it work" loop that desktop tools make
   ceremonial.
2. **First-time designers** — people who have never made a font but want to
   try. Most enter through "Open the example project." Need a clear path in
   without being talked down to.

The interface optimizes for solo designers' taste while remaining inviting to
first-timers at the moments of first contact (home, Welcome modal, Brief tab).

---

## Brand Personality

**Foundry. Quiet. Considered.**

This is a type foundry's editor, not a software dashboard. The product has the
voice of a small foundry that takes typography seriously and treats the user
as a fellow practitioner.

### Voice signals already established
- **Time-of-day greetings** on the home page ("Design your own typeface, quietly,
  in the morning")
- **Per-milestone copy** at glyph counts 1 / 5 / 10 / 25 / 50 / 100 / 200
  ("One. The foundry begins." → "200. Few personal projects ever reach here.")
- **Hoefler Text serif** on identity moments: home tagline, project name input,
  glyph mark in the OG image, the 404 page heading ("Not in this typeface.")
- **`tracking-wider uppercase` mono labels** for category eyebrows (FONT STUDIO,
  COMPILED, BRIEF)
- **Restrained celebration**: confetti only on real ship moments (first glyph
  drawn, audit clean, release sealed), never on routine saves or interactions
- **ASCII art banner** in the dev console (Tre logo, restrained foundry palette)
- **Konami code → "Foundry mode unlocked"** — Easter egg, not a feature

### What "foundry, quiet, considered" rules out
- Loud success states (no full-screen celebrations, no "Yay!")
- Marketing-style headlines on in-app surfaces
- Over-explanation of tools (assume the user is competent)
- Emoji decorations (occasional restrained · ★ ◆ ・ is fine)

## Aesthetic Direction

### Theme
**Light is primary.** Cream canvas (`hsl(240 6% 97%)`), warm surface, ink-dark
foreground. Dark mode is supported and equally considered — never the default,
never the "designed first" mode. The brand reads in either, but light is the
foundry's day-shift.

### Typography
- **Display + identity** (large headers, hero tagline, project name, glyph
  thumbnails in the OG image, the 404 "Not in this typeface."):
  `Hoefler Text, ui-serif, Georgia, serif`. Used sparingly — it's the foundry's
  signature, not the workhorse.
- **Body / chrome** (sidebars, labels, buttons, body text, tab nav):
  system-ui sans. `font-feature-settings: 'cv11', 'ss01'` enabled.
- **Numeric / data** (`data-numeric` attribute): `ui-monospace, 'SF Mono', Menlo`.
  Used for codepoints (U+0041), counts (8/26 drawn), version numbers, byte sizes.
  Never as a "this is technical" decoration — only on actual numeric data.
- **No italics in UI chrome.** Italic is reserved for body prose if anywhere.
  For UI emphasis, mix families + weights instead.

### Color
- **Tokens** via `hsl(var(--*))` in `src/app.css`, with both light and dark
  themes + a manual `theme-dark` class override.
- **Brand accent**: `--accent: 222 88% 55%` (foundry blue). Used sparingly —
  selected tab, primary CTA, focus rings, anchor link hover.
- **`-strong` companions** for text on tinted backgrounds:
  `--accent-strong: 222 88% 38%`, `--warn-strong: 35 90% 28%`, etc. Always
  use these for body text on `bg-X-soft` or `bg-X/10/15/20` — bare `text-X`
  is reserved for icons (3:1 non-text threshold).
- **Forbidden palettes**: cyan-on-dark, purple-to-blue gradients, neon
  accents. No gradient text under any circumstance. No `text-fg-subtle/N`
  opacity stacks (caught + fixed in `f13b1ff`).

### Spacing & layout
- **Fluid spacing where it earns it** (home hero, in-app header dual rows).
  Standard tab content uses generous breathing room over density.
- **Asymmetric compositions** preferred where they reinforce hierarchy. The
  home page has rhythm via varied card sizes, not uniform grids.
- **No cards-in-cards.** Flatten the hierarchy. The audit page's filter pills
  are inside a Panel; the issues list flows beneath, not inside more cards.
- **Subtle borders** (`hsl(240 6% 88%)`) preferred over heavy shadows. Where
  shadows appear (hover lifts on project cards), they're soft and short.

### Anti-references — what this is explicitly NOT
- **Glyphs / FontLab pro-tool density** — Font Studio is not competing on
  toolbar walls, infinite docks, or tiny labels. The pro-density user is
  served by their desktop tool; Font Studio is the browser-first companion.
- **The "AI app" look** — cyan-on-dark, purple-blue gradients, glassmorphism,
  glowing accents, neon. Reads as bot-made. Forbidden.
- **Generic SaaS dashboard** — repeating card grids of stat-with-up-arrow,
  big gradient header, sidebar with icons + labels. The "every Linear clone"
  template.

The 404 page (`src/routes/+error.svelte`) shows the right tone:
"Not in this typeface." with the HTTP status displayed as `U+0404` in the
tofu-box metaphor that type designers will recognize. Reference for what the
voice should sound like across the product.

## Design Principles

These five derive from the audience + personality + aesthetic decisions
above. They take precedence over taste when in conflict.

### 1. Restraint over excitement
Every visual choice should feel considered, not loud. Confetti fires only on
real ship moments. Toasts whisper "Saved." not "✓ Your changes were
successfully persisted!". When in doubt, do less.

### 2. Typography is the primary visual element
The product is a type editor. Type carries identity, not decorative graphics.
Hoefler serif at identity moments, sans for chrome, mono for numeric data —
that's the entire palette. No illustrations competing with type, no
gradient text "for impact," no monospace-as-tech-shorthand.

### 3. Welcome first-timers without condescension
The Welcome modal is short, the Brief tab has a one-line subhead, the "Open
the example project" CTA is a one-click on-ramp. Nowhere does the product
explain what a glyph is — it assumes the user is here to learn by doing.
Friendly is fine; patronizing is not.

### 4. Every contrast surface passes WCAG-AA
Body text on tinted backgrounds uses the `-strong` token variants. The
`/_axe-core` suite catches regressions on serious/critical violations
across 15 surfaces. New text on a tinted bg → use the `-strong` companion.
New form input → has an aria-label.

### 5. No AI-slop fingerprints
Specific things this product will never have:
- Gradient text
- Glassmorphism / blur cards / glow borders as decoration
- Monospace fonts used to "feel technical"
- Pure black `#000` or pure white `#fff` (all neutrals tinted toward the
  brand hue)
- Big icons with rounded corners above every heading
- Repeating card grids of identical-sized stat boxes
- Generic empty-state illustrations
- Cyan / purple / neon accents on dark backgrounds

When designing a new surface: review this list before shipping. If any
fingerprint is present, redesign.
