# Demo-font quality pass — 2026-06-10 ("make the demo font amazing")

The in-editor demo font (Studio Geometric) was half rectangles: `n/m/h/u`
were stacked boxes, `s` was a five-bar digital digit, `c` a square
bracket, `a` four rectangles, `e/b/d/p/q/g/ß` 16-gon polygon rings.
Worse: the homepage "specimen" wasn't even rendering the demo font — a
Swiss-migration override (`[style*='Hoefler'] { font-family … !important }`)
matched the specimen's fallback chain and silently forced IBM Plex
since the Swiss pass.

## What changed (src/lib/font/demo-project.ts)
- New primitives: `lcRing` (overshot ring, optically-corrected walls
  94/84 vs the 90u stem), `lcArch` (single-contour n-skeleton ported
  from the static script's documented recipe), `bowlWithStem` (fused
  bowl+stem outline via `ellipseArc` — no overlapping pieces),
  `rotate180`, `cutRect`, `ellipseHalfWidthAt`.
- Redrawn: o e a c s b d p q g n m h u r f w + fixed z (abutting
  diagonal), k (self-crossed lower leg), y/v/w/x/R-leg directions.
- Every 16-gon ring font-wide swept to true kappa cubics (S Q ß Ф °
  digits 0/6/8/9 æ œ © ® @ …).
- All solid contours positive, counters negative (path.ts vocabulary:
  positive = 'ccw'); overlaps eliminated where direction mixing made
  renderers disagree.
- Italic master inherits everything (programmatic shear).

## Single source of truth for the marketing font
`scripts/build-demo-otf.mjs` (+ `scripts/src/build-demo-otf-entry.ts`)
now generates `static/demo-fonts/StudioGeometric-Regular.otf` FROM the
demo project via the editor's own `buildFont` — the hand-drawn
duplicate in build-demo-fonts.mjs had drifted (its s was rectangles
while the editor improved). `npm run build:demo-fonts` runs both; the
unified OTF wins. The old script still makes StudioSlab (and got the
rounded S/s + fused bowls ported anyway).

## Verified
- Exported OTF specimen at 170px: Patens / Hamburgevons / full
  lowercase read as a real condensed geometric (screenshots in
  /tmp/patens-qa/specimen-otf2.png, home-specimen-unified.png).
- Stem rhythm: median 90, real outliers fixed (was 6/12 — b/d 80,
  p/q 73, l/i 69; now 3/12, all measurement artifacts on plain rects).
- Gate: svelte-check 0/0, lint clean, 796 unit (auto-kern snapshot
  intentionally updated — real silhouettes produce richer suggestions),

## Notes for future glyph work
- The renderer family disagrees on overlapping contours when directions
  mix — keep every solid positive, counters negative, and prefer fused
  single outlines or exact abutment over overlaps.
- OG cards (/og/*) render from the demo OTF and pick the new font up
  automatically on deploy.
- Remaining taste pass (Alejandro): c aperture width, s spine weight,
  f/g hook depth — all parametric one-liners now.

## Design revision — squared geometric ("Eurostile pass", same day)

Direction from Alejandro: the circular forms still read amateur — go
SUPER GEOMETRIC, SQUARED. Implemented as a parametric rounded-rect
system in demo-project.ts (`rr` per-corner-radius primitive, `sqRing`,
`sqArch`, `sqC`, constants SQ_R/SQ_RI/SQ_RS/SQ_R_CAP): every bowl,
arch, hook and counter is now a rounded rectangle — Eurostile/
Microgramma genre, which also unifies with the already-rectangular
caps. Rebuilt: all bowls + arches lowercase, squared s (both cases),
caps O Q C G D B P R S U W K, digit 0. Fixed long-standing cap bugs
exposed by the pass (U's notched bowl, W's crossed arms, K's bowtie
leg). No external font used as a base — keeps "drawn in Patens" true.
Old circular helpers deleted; auto-kern snapshot re-baselined; gate
green (796 unit, 83 e2e). Full alphabet render:
/tmp/patens-qa/sq-all2.png. Taste knobs (one-liners): SQ_R (110) for
how round the squares are, c/e aperture sizes, S middle-bar weight.
